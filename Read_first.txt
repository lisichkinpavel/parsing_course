По поводу уникальных вакансий. В ссылках на вакансии есть id, например "https://hh.ru/vacancy/38033348?query=%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BC%D0%B8%D1%81%D1%82".
его я и принимал как признак уникальности.
Я добавил в словарь с вакансиями поле "_id" с этим идентификатором, и потом добавлял в БД документы с этими id. Если возникает исключение и в базе уже есть документ с этим id, то этот документ отрбасывается.

На запрос hh отдает максимум 2000 вакансий(40 страниц по 50 вакансий). Я сделал два раза запрос по одним параметрам и получил "Уникальные вакансии, записанные в БД: 2161". То есть за 2 прохода по 2000 вакансий, уникальными из них оказались только 2161.
